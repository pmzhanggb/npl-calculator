# 信用卡不良债权包投资测算 (Credit Card NPL Bulk-Transfer Yield Model)

> 📌 本项目已纳入 Git 版本管理（`main` 分支，根目录运行 `git log` 查看历史）。

## 项目目标

为不良资产投资业务的**投委会决策**提供一套可在浏览器中即时测算的工具：给定一个信用卡不良债权包的处置假设（季度回收节奏、资金结构、催收成本、AMC 通道费/管理费、返点等），反推：
- 劣后资金的 **IRR / MOIC / 累计收益** 是否达到目标；
- 在目标 IRR 下的**最高可买价格率**（议价上限）；
- 整体回收需要缩放多少倍才能**盈亏平衡**；
- **优先级资金（AMC 配资 + 其他融资）的覆盖倍数**与清偿状态。

从而对"是否买包、按什么价格买"给出可量化的判断。

## 技术栈

纯静态前端，零依赖、零构建：
- `网页/index.html` — 单页结构，含左侧参数面板 + 右侧分析面板
- `网页/app.js` — 全部测算逻辑、IRR 求解、Chart 绘制（Canvas 原生 API）
- `网页/styles.css` — 样式（CSS 变量 + Grid 布局，自适应至 720px）
- 数据持久化：**LocalStorage**（参数快照历史，最多 30 条，跨刷新/跨会话保留）
  - 存储键：`npl_model_history_v1`
  - 写入失败兜底：`alert` 提示，不影响当前测算

## 测算周期与口径

- **季度结算**：5 年共 **20 个季度**（`QUARTERS = 20`，`QUARTERS_PER_YEAR = 4`）
- **回收口径**：按各年季回收率 × 季初资产本金计提
- **利率口径**：年利率 / 4 = 季利率
- **IRR 复利**：季度复利，年化 = (1 + 季IRR)⁴ − 1
- **运营成本**：字段保留"元/月"语义，季度结算时 × 3

### 2. 测算结果（顶部 7 个指标卡）

- **投包结论** — 三档：`可以买` / `可谈价` / `不建议`，按"IRR 达标 + 劣后盈利 + 优先级清偿"组合判定

- **最高可买价格率** — 在 `targetIrr` 下二分搜索得到的购包价上限（折数显示）
- **盈亏平衡回收倍率** — 让劣后盈亏平衡时，整体回收率的整体缩放系数
- **优先级覆盖倍数** — 优先级本息累计兑付 / 优先级本金义务
- **盈亏平衡时点** — 三参与方各自回本季度：`劣后 Q? / AMC Q? / Mezz Q?`，未回本显示"未回本"

## 测算核心逻辑（`project()` in app.js）

按季度循环 20 期，每期 5 步：

1. **回收**：`gross = min(资产本金, 季初本金 × 该年季回收率)`
2. **成本扣减**：`可分配额 = max(0, 回收 − 催收费 − 诉讼费)`
3. **优先级兑付**（先 AMC 后 Mezz）：先付当期利息，再偿本金，直到该级清偿或可分配额用尽
4. **劣后**：`(剩余可分配额 − 季运营 − 季管理费) + 当季返点` 计入劣后现金流
5. **AMC 收益归集**：当季 AMC 利息 + 当季管理费（首季再加一次性通道费）

**一次性扣减**（购包时）：
- 劣后初始投入 = 自有资金
- 通道费 = `amcChannelFixed + 购包价 × amcChannelRate`，在 t=1 一次性从劣后支出，并计入 AMC 累计收益
- 体现为 IRR 现金流：t=0 投入 → t=1 通道费 → t=1..20 季度回收（共 22 期）

辅助算法：
- `findMaxDiscount()` — 二分反推满足 `targetIrr` 的最大购包价率
- `findBreakEvenMultiplier()` — 二分反推劣后盈亏平衡的回收倍率
- `annualizedIrr()` — 网格 + 二分混合求根器，1800 步扫描 + 80 次二分精修，结果按季度复利年化
- **盈亏平衡时点**：主循环中累计跟踪 `cumulativeAmcPaid` / `cumulativeMezzPaid`，首次满足 `累计兑付 ≥ 配资本金` 的季度即为该参与方盈亏平衡时点

## AMC 与 Mezz 收费模型

| 项目 | 收取方式 | 计入方 | 扣减方 |
| --- | --- | --- | --- |
| AMC 通道费（固定 + 对价 × 费率） | 一次性 | AMC 收益 | 劣后（首期） |
| AMC 管理费（% 回收额） | 每季 | AMC 收益 | 劣后 |
| AMC 利息（按余额计提） | 每季 | AMC 收益 | 优先级兑付流（先于劣后分配） |
| **Mezz 利息（按原始本金固定计提）** | **每季刚性兑付** | **Mezz 收益** | **劣后现金流（与回收额无关）** |
| Mezz 本金偿还 | 随可分配额兑付 | Mezz 累计兑付 | 可分配额 |

> Mezz 模型按"明股实债"处理：
> 1. **利息每季固定**：`funds.mezz × mezzRate/4`，不随本金偿还递减（从 T0 起算）
> 2. **劣后刚性兑付**：利息由劣后从现金流直接扣除，与当季可分配额多少无关
> 3. **Mezz 本金兑付**：仅从可分配额中按"先还本"顺序兑付（不再"先息后本"）
> 4. **Mezz 累计曲线**：累计利息 + 累计本金偿还 = 总兑付额
> 5. **Mezz 盈亏平衡时点**：累计本金偿还 ≥ 配资本金
> 6. **劣后 IRR 影响**：因每季固定承担 Mezz 利息，IRR 会显著低于"利随本清"模型
>
> 本金偿还不计入 AMC 累计曲线（按"不需要看贷款本金"的口径）；Mezz 因为劣后刚性兑付利息，本金偿还计入 Mezz 累计曲线（展示口径不同）。
